# 如何确保Rules被遵守

## 问题背景

即使Rules已加载到Cursor中，AI仍可能由于习惯性行为而违反规则，特别是`decision-change-approval`规则（在聊天中列选项而不是使用AskQuestion工具）。

## 四层强化机制

我们实施了4层机制来确保Rules被遵守：

### 第1层：增强Rules的强制性 ✅

**文件**: `00-CORE-ENFORCEMENT.mdc`

**改进**:
- 在description中加入多个🔴和CRITICAL、MANDATORY关键词
- 在规则开头明确标注"CRITICAL"
- 特别强调最容易违反的规则（AskQuestion）
- 增加详细的违规示例

**效果**: 提高AI对规则的注意力

### 第2层：创建Rules监督机制 ✅

**文件**: `rules-self-check.mdc`

**机制**:
- 强制要求在thinking中执行自检流程
- 4步检查法：识别场景 → 匹配规则 → 执行检查 → 声明遵守
- 特别监督AskQuestion规则
- 违规自动纠正流程

**效果**: 形成系统化的自我监督

### 第3层：项目级强化提醒 ✅

**文件**: `.cursorrules` (每个项目)

**作用**:
- 作为全局Rules的补充提醒
- 突出显示最容易违反的规则
- 提供简化的自检清单
- 说明如何反馈违规

**效果**: 双重保险，项目级提醒

### 第4层：持续反馈和训练 ✅

**方法**:

1. **立即指出违规**
   当AI违反规则时，立即明确指出：
   ```
   "你违反了decision-change-approval规则！
   你在聊天中列出了选项，应该使用AskQuestion工具。"
   ```

2. **要求thinking反思**
   要求AI在下一次回复的thinking中：
   - 承认违规
   - 分析原因
   - 说明如何纠正
   - 承诺改进

3. **正面强化**
   当AI正确遵守规则时，可以表扬：
   ```
   "很好！你正确使用了AskQuestion工具。"
   ```

4. **记录模式**
   如果某条规则反复被违反：
   - 进一步增强该规则的描述
   - 在thinking自检中加入特别检查
   - 考虑修改规则的表达方式

## 使用指南

### 对于用户

**当AI违反规则时**:
1. 立即指出："你违反了[规则名称]"
2. 说明应该怎么做："应该使用AskQuestion工具"
3. 等待AI在thinking中反思
4. 确认AI理解并纠正

**当AI遵守规则时**:
1. 可以简单表扬（可选）
2. 继续正常对话

### 对于AI

**每次回复前必须**:
1. 在thinking中执行rules-self-check流程
2. 特别检查是否需要用AskQuestion
3. 声明遵守哪些规则
4. 如有违规立即纠正

**违规后必须**:
1. 在thinking中承认违规
2. 分析为什么违规
3. 说明如何纠正
4. 重新执行正确操作

## 效果验证

### 验证方法

1. **测试AskQuestion**:
   ```
   用户："给我一个选项测试"
   AI应该：使用AskQuestion工具
   ```

2. **测试零臆想**:
   ```
   用户："这个表应该有什么字段？"
   AI应该：先查询表结构，不假设
   ```

3. **测试禁止报告**:
   ```
   用户："完成任务"
   AI应该：在回复中总结，不生成报告.md
   ```

### 预期效果

- ✅ AI在thinking中主动执行自检
- ✅ 遇到多方案自动使用AskQuestion
- ✅ 违规后能自我发现并纠正
- ✅ 违规率逐步降低

## 持续改进

### 监控指标

1. **违规频率**: 每个规则的违反次数
2. **自我纠正率**: 违规后AI自己发现的比例
3. **thinking质量**: 是否包含完整自检流程

### 优化建议

1. 如果某规则反复违反：
   - 增强该规则的CRITICAL标记
   - 在rules-self-check中加入专门检查
   - 在.cursorrules中突出提醒

2. 如果整体违规率高：
   - 考虑简化rules数量
   - 合并相似规则
   - 优化规则表达

3. 如果thinking检查流于形式：
   - 增加具体检查点
   - 要求明确声明遵守情况
   - 加强违规后的反思要求

## 常见问题

**Q: AI还是会忘记使用AskQuestion怎么办？**
A: 
1. 每次违规立即指出
2. 检查thinking中是否执行了自检
3. 如果没有，要求重新思考并使用工具
4. 多次反馈后会形成习惯

**Q: 规则太多AI记不住？**
A:
1. 00-CORE-ENFORCEMENT已经是精简版
2. rules-self-check提供系统化流程
3. .cursorrules提供项目级提醒
4. 不需要记住所有细节，只需执行自检流程

**Q: 如何知道规则是否生效？**
A:
1. 问AI："请列出当前上下文中的rules"
2. 测试关键规则（AskQuestion、零臆想等）
3. 检查thinking中是否包含自检流程

## 总结

通过4层机制的配合：
1. **强制性**（加强规则本身）
2. **监督性**（系统化自检）
3. **提醒性**（项目级强化）
4. **反馈性**（持续训练）

可以显著提高Rules的遵守率。关键是**持续反馈**和**系统化自检**。
